# Demo Storyboard

Reference artifact: the committed pressure plot in `assets/neighborhood-pressure-review-live.png`, generated by the package demo CLI from mixed point, line, and polygon features.

## 1. Frame the use case

Introduce the repo as the custom spatial package lane: not another one-off analysis, but a package users can import and build on.

## 2. Show the frame object

Open `src/geoprompt/frame.py` and show that the package exposes a `GeoPromptFrame` with geometry-aware methods such as `centroid`, `distance_matrix`, `geometry_lengths`, `geometry_areas`, and `neighborhood_pressure`.

## 3. Show the equations

Point out that the package-level building blocks now include decay, influence, interaction, corridor strength, and area similarity.

## 4. Run the demo

Generate `outputs/geoprompt_demo_report.json`, then open `assets/neighborhood-pressure-review-live.png` as the committed real visual. Call out that the plot is drawing multiple geometry types but still ranking them through one package API.

## 5. Explain the extension path

Close by noting that the current package is deliberately small, but already structured for later geometry types, CRS support, joins, overlays, or richer spatial operators.
