# Publishing Guide

## Recommended Standalone Repository Name

- experience-builder-station-brief-widget

## Recommended Description

- React and TypeScript GIS widget prototype inspired by Experience Builder patterns for station filtering, operational summaries, and selection-driven detail panels.

## Suggested Topics

- react
- typescript
- gis
- geospatial
- frontend
- vite
- experience-builder

## Split Steps

1. Create a new empty repository named `experience-builder-station-brief-widget`.
2. Copy this project folder into the new repository root.
3. Preserve `src/`, `tests/`, `assets/`, `docs/`, and the frontend build files.
4. Use `assets/widget-live-screenshot.png` as the initial public visual, and add a real GIF or browser capture if you want a second visual.
5. Reference `docs/architecture.md`, `docs/demo-storyboard.md`, and `docs/site-map.md` from the README when polishing the public pitch.

## First Public Polish Pass

- Add a short note explaining how the prototype maps to real Experience Builder widget responsibilities.
- Keep the public-safe disclaimer near the top of the README so the repo does not imply product screenshots or private code access.
- Add a short GIF showing the filter, selection, and detail-panel flow in the browser build.
