# Case Study Walkthrough

## Purpose

This walkthrough mirrors the structure of the original flood-forecasting article while using the reproducible artifacts generated by this repository.

## 1. Study Frame

The original goal was to explore whether wavelet-denoised autoregressive forecasting could improve short-horizon flood-stage prediction. This public version keeps that method question intact while using South Texas analog gauges that can be refreshed directly from USGS.

Primary review artifacts:

- [../outputs/review-summary.html](../outputs/review-summary.html)
- [../outputs/arroyo_flood_forecast_report.json](../outputs/arroyo_flood_forecast_report.json)

## 2. Study Area Analog

The primary series comes from Nueces River at Calallen, TX (`08211500`), which functions here as a reproducible South Texas analog rather than a claim that the data is Arroyo Colorado telemetry.

The secondary series comes from Oso Creek at Corpus Christi, TX (`08211520`) to test whether the method behaves differently on a noisier signal.

Reference artifacts:

- [../data/arroyo_stage_series.json](../data/arroyo_stage_series.json)
- [../data/oso_creek_stage_series.json](../data/oso_creek_stage_series.json)

## 5. Raw Versus Denoised Holdout Review

The holdout chart is the central model-comparison view. It shows the observed holdout sequence, the best raw forecast, the best denoised forecast, and the Monte Carlo median with the P10-P90 band.

Reference artifact:

- [../outputs/charts/holdout-forecast-comparison.png](../outputs/charts/holdout-forecast-comparison.png)

For the current Nueces analog, the raw branch wins on holdout PMSE. That is useful because it demonstrates the workflow is evaluating rather than assuming denoising always improves performance.

## 6. Scenario And Threshold Review

The Monte Carlo exceedance chart turns the deterministic forecast into an uncertainty-aware review artifact.

Reference artifact:

- [../outputs/charts/threshold-exceedance-probability.png](../outputs/charts/threshold-exceedance-probability.png)

The threshold used here is a reproducible percentile-based review threshold rather than an asserted official flood stage for Arroyo Colorado.

## 7. Cross-Site Comparison

The second gauge makes it possible to ask a more interesting question: does wavelet denoising help more on a noisier series?

Reference artifacts:

- [../outputs/cross-site-comparison.html](../outputs/cross-site-comparison.html)
- [../outputs/charts/wavelet-benefit-comparison.png](../outputs/charts/wavelet-benefit-comparison.png)
- [../outputs/multi_site_comparison.json](../outputs/multi_site_comparison.json)

Current result:

- Nueces (`08211500`) is smoother and the raw branch wins.
- Oso Creek (`08211520`) is noisier and the denoised branch wins.
- In this sample, denoising helps more on the noisier site.

## 8. Main Takeaways

- The workflow preserves the method logic of the original study in a Python package rather than MATLAB.
- The project now produces both software-style artifacts and article-style presentation material.
- The second-gauge experiment strengthens the portfolio value because it shows comparative reasoning, not just a single-case demonstration.

## 9. Reproducing The Walkthrough

Run the main workflow:

```bash
arroyo-flood-lab
```

Refresh the public data inputs when needed:

```bash
refresh-arroyo-flood-data
python -m arroyo_flood_forecasting_lab.refresh_data --site-code 08211520 --output-path data/oso_creek_stage_series.json
```
