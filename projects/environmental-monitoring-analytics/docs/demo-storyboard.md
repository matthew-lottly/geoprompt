# Demo Storyboard

## Intended Audience

- Reviewers looking for analytical depth beyond backend APIs
- Teams that care about SQL fluency and concise reporting

## Narrative Arc

1. Start with the generated monitoring brief and explain that the project is designed to produce review-ready operational output.
2. Point out alert rate, regional pressure, and latest alert stations as decision-support metrics.
3. Show that the implementation is SQL-first through DuckDB rather than notebook-only analysis.
4. Position the project as the analytics companion to the API repo.

## What Reviewers Should Notice

- The work is output-oriented, not just exploratory.
- The metrics are operational and easy to understand.
- The project complements the API and warehouse repos as part of one coherent monitoring theme.
